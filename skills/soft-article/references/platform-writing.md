# 平台差异化写作指南

## 核心原则

**同一主题，三篇不同文章**：不是改改标题发多处，而是从不同角度、用不同结构、针对不同读者重新写。

---

## 掘金：技术深度优先

**读者画像**：前端/后端开发者，追求技术细节和最佳实践

**写作重点**：
- 70% 代码 + 30% 文字
- 完整可运行的代码示例（带注释）
- 技术选型理由、性能对比、踩坑经验
- 框架关键词（React/Vue/Next.js/FastAPI）

**标题模板**：
- 《用 [框架] 实现 [功能]：完整代码 + 最佳实践》
- 《[技术] 深度解析：从原理到实战》
- 《[场景] 性能优化：3 个关键技巧》

**结构**：
```
# 标题
## 背景（为什么需要这个技术）
## 技术选型（对比 2-3 个方案）
## 完整实现（代码 + 注释）
## 性能优化（具体数据对比）
## 踩坑记录（3-5 个常见问题）
## FAQ
## 总结（技术要点 + GitHub 链接）
```

**产品植入方式**：
- 代码里的 `base_url` 自然用 xingjiabiapi.org
- 性能对比表里加一列"使用中转服务"
- 踩坑记录里提"官方 API 国内访问不稳定，改用中转服务解决"

## CSDN：教程步骤优先

**读者画像**：初学者、搜索引擎来的用户，需要手把手教程

**写作重点**：
- 步骤编号清晰（第一步、第二步...）
- 每步都有截图或代码
- 常见错误和解决方案
- SEO 关键词密度高（标题+小标题+正文）

**标题模板**：
- 《[技术] 完整教程：从安装到部署（2026 最新版）》
- 《5 分钟搞定 [功能]：保姆级教程》
- 《[问题] 解决方案：3 种方法对比》

**结构**：
```
# 标题（含年份+关键词）
## 前言（为什么写这篇教程）
## 环境准备
### 第一步：安装依赖
### 第二步：配置 API
### 第三步：编写代码
### 第四步：运行测试
## 常见问题
### 问题1：xxx 报错
### 问题2：xxx 不生效
## 成本对比（表格）
## 总结（完整代码 + 推广信息）
```

**产品植入方式**：
- 第二步"配置 API"里自然用 xingjiabiapi.org
- 成本对比表里详细列出官方 vs 中转服务价格
- 常见问题里提"国内访问慢？用中转服务解决"

## 头条：科普价值优先

**读者画像**：泛科技用户，对 AI 感兴趣但不懂技术

**写作重点**：
- 80% 科普知识 + 20% 产品
- 用类比、故事、场景化解释技术概念
- 少代码多图表，多用"是什么""为什么""怎么做"
- 去掉产品名文章还有独立价值

**标题模板**：
- 《[技术] 是什么？5 分钟入门指南》
- 《大模型 API 怎么用？看完这篇就懂了》
- 《[场景] 省钱攻略：3 个实用方法》

**结构**：
```
# 标题（科普向，不用标题党）
## 什么是 [技术]？（用类比解释）
## 为什么需要 [技术]？（3 个实际场景）
## 怎么用 [技术]？（简单代码示例）
## 主流方案对比（表格，多家对比）
## 成本怎么算？（具体例子）
## 国内怎么用？（解决方案）
## 常见问题（FAQ）
## 总结（核心要点）
```

**产品植入方式**（最隐蔽）**：
- 只在代码示例的 `base_url` 里出现一次
- 成本对比表里和官方+其他 2-3 家中转站一起对比
- 不能有微信/邮箱/重复链接
- 个人主页放联系方式，正文不放

## 知乎：观点深度优先

**读者画像**：高知用户，喜欢深度分析和独立观点

**写作重点**：
- 问答风格，有明确观点
- 个人经历 + 数据支撑
- 多角度分析，不偏颇
- 少代码多分析，FAQ 段落必须有

**标题模板**：
- 《[问题]？我用了 3 个月后的真实体验》
- 《关于 [技术]，这 5 个坑你一定要知道》
- 《[对比]：我测试了 10 个方案后的结论》

**结构**：
```
# 标题（问答式）
## 直接回答问题（一句话结论）
## 我的使用背景（个人经历）
## 详细分析（3-5 个维度）
## 数据对比（真实测试结果）
## 踩坑经验（3-5 个）
## FAQ（结构化问答）
## 总结（观点 + 建议）
```

**产品植入方式**：
- 个人经历里自然提"我用的是 xingjiabiapi.org"
- 数据对比表里和官方+其他方案一起对比
- 不能有微信/邮箱，只在个人简介放
- 评论区可以补充链接（但不能每篇都补）

**读者画像**：年轻用户，对 AI 感兴趣但不懂技术

**写作重点**：
- 800-1200 字，科普向
- 用类比、故事、场景化解释技术概念

**标题模板**：
- 《[技术] 是什么？5 分钟入门指南》
- 《大模型 API 怎么用？看完这篇就懂了》
- 《[场景] 省钱攻略：3 个实用方法》

**结构**：
```
# 标题（≤20字，科普向）
## 什么是 [技术]？（用类比解释）
## 为什么需要 [技术]？（3 个实际场景）

## 主流方案对比（表格，多家对比）
## 成本怎么算？（具体例子）
```

**产品植入方式**：

## 公众号：故事感优先

**读者画像**：订阅用户，愿意看长文

**写作重点**：
- 800-1500 字，有故事线
- 具体人物/场景/对话
- 轻松易读，有温度
- 结尾引导关注/转发

**标题模板**：
- 《[场景]：一个开发者的真实经历》
- 《我是如何用 [技术] 解决 [问题] 的》
- 《[对比]：3 个月使用体验分享》

**结构**：
```
# 标题（故事感）
## 开头（场景描述 + 痛点）
## 我的尝试（时间线叙述）
## 转折点（发现解决方案）
## 具体效果（数据 + 体验）
## 经验总结（3-5 条）
## 结尾（引导 + 推广信息）
```

**产品植入方式**：
- 故事里自然提"后来朋友推荐了 xingjiabiapi.org"
- 文末完整推广信息（网站+微信+邮箱）
- 可以加二维码图片

## 实战案例：同一主题的 3 种写法

**主题**：Claude API 成本优化

### 掘金版（技术深度）
标题：《Claude API 成本优化实战：3 个技巧降低 70% 开销》
重点：代码示例（缓存、prompt 优化、模型选择）+ 性能数据对比

### CSDN 版（教程步骤）
标题：《Claude API 省钱完整教程：从 $150/月降到 $45/月（2026 最新）》
重点：第一步配置、第二步优化、第三步监控，每步有截图

### 头条版（科普价值）
标题：《大模型 API 怎么省钱？3 个方法帮你降低 70% 成本》
重点：什么是 token、为什么贵、怎么省钱（通俗易懂）

- [ ] 三篇文章读者不同（开发者/初学者/泛用户）
- [ ] 产品植入方式不同（代码/对比/体验）
- [ ] 去掉产品名每篇都有独立价值
